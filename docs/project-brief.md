# Project Brief

## Working Title

**Agentic Patterns: A Pattern Language for Human-Guided AI Work**

This is the recommended public title for now. It keeps `Agentic Patterns` as the memorable project
name while making the deeper ambition clear.

## Alternate Titles

- **Agentic Patterns: Reusable Context for Human-Guided AI**
- **Agentic Patterns: A Catalog of Prompts, Context, and Harnesses**
- **The Agentic Pattern Language**
- **Agentic Patterns: Reusable Knowledge for AI-Assisted Work**

The first title is strongest because it avoids reducing the project to prompts. The project is
really about the language, context, tools, and protocols humans use to direct AI agents.

## One-Sentence Description

Agentic Patterns is an open catalog and pattern language for reusable prompts, context structures,
harnesses, role contracts, and protocols that help humans guide AI agents through repeated
creative, technical, and knowledge work.

## Short Description

Agentic Patterns collects repeatable solutions to recurring problems in human-guided AI work.
Agentic coding is the first mature domain because it exposes clear context files, tools, tests,
repositories, review loops, and durable artifacts. The same pattern language also applies to image
generation, literature and writing, product design, design-to-code, video/media production,
research, analysis, and other domains where humans guide AI through repeated workflows.

It borrows the pattern-language idea from software engineering and adapts it to prompts, context
engineering, review loops, multi-agent orchestration, role contracts, and tool-use protocols. The goal is
to make effective human-AI workflows easier to name, teach, critique, automate, and improve.

For the intellectual background, see [Pattern Language Roots](foundations/pattern-language-roots.md).

## Why It Matters

AI agents can match enormous numbers of patterns, but they still need human goals, judgment,
memory, and abstraction. Humans often discover practical workflows before they can formally prove
why those workflows work. Naming those workflows as patterns makes them easier to communicate,
test, refine, and reuse.

This project treats patterns as memetic units of operational knowledge: compact, transmissible
solutions that help humans and cognitive systems coordinate around common goals.

## Project Scope

In scope:

- Prompt and instruction patterns.
- Context packet and memory patterns.
- Planning, execution, review, and verification harnesses.
- Multi-agent orchestration patterns.
- Tool-use patterns.
- Pattern schemas and agent-readable indexes.
- Field notes and evidence from agentic coding, creative generation, design, writing, research, and
  other repeated human-AI workflows.

Out of scope for the seed stage:

- A full hosted pattern registry.
- Benchmarks that claim universal ranking of patterns.
- Vendor-specific marketing content.
- Generic AI news or unsorted prompt collections.

## Initial Audience

- Developers practicing agentic coding.
- Designers, writers, researchers, artists, product builders, and media creators using generative
  or agentic AI workflows.
- People writing Claude Code, Codex, Gemini, or similar agent instructions.
- Maintainers documenting local agent workflow practice.
- Researchers and practitioners studying human-AI software work.

## Seed-Stage Success Criteria

- The repository explains the project clearly enough for public GitHub release.
- At least three patterns demonstrate the desired quality bar.
- New contributors can add a pattern without asking for hidden context.
- Agents can read `AGENTS.md` and help maintain the catalog consistently.

# Licensing

This project uses a split license.

## Recommended Direction

Use a dual-license model:

- Pattern catalog, documentation, prompts, and field notes: **Creative Commons Attribution 4.0
  International (CC BY 4.0)**.
- Code, schemas, and approved software artifacts: **MIT**.

## Why This Fits

The catalog is knowledge work. CC BY 4.0 encourages reuse, adaptation, translation, and citation
while preserving attribution.

Software artifacts use MIT because it is simple, familiar, permissive, and easy for contributors to
understand.

## Alternative

Use Apache-2.0 for software if explicit patent language is important.

## Repository Files

- `LICENSE`: split-license summary and contribution terms.
- `LICENSE-MIT`: MIT license text for software artifacts.
- CC BY 4.0 is referenced by URL: https://creativecommons.org/licenses/by/4.0/

Before publishing, make sure contributors understand that submitted patterns and field notes are
accepted under the repository license.

# プロジェクト概要

## 作業タイトル

**Agentic Patterns: A Pattern Language for Human-Guided AI Work**

現在の公開タイトルとして最も適切な候補です。`Agentic Patterns` という短く記憶しやすい
名前を維持しながら、より大きな目標を伝えます。

## 代替タイトル

- **Agentic Patterns: Reusable Context for Human-Guided AI**
- **Agentic Patterns: A Catalog of Prompts, Context, and Harnesses**
- **The Agentic Pattern Language**
- **Agentic Patterns: Reusable Knowledge for AI-Assisted Work**

最も強い方向は最初のタイトルです。このプロジェクトは単なるプロンプト集ではなく、
人間が AI エージェントを指揮するために使う言語、コンテキスト、ツール、プロトコルに
関するものです。

## 一文説明

Agentic Patterns は、反復する創造的・技術的・知識的作業において人間が AI エージェントを
より良く導けるよう、再利用可能なプロンプト、コンテキスト構造、ハーネス、ロール契約、
プロトコルをまとめた公開カタログ兼パターン言語です。

## 短い説明

Agentic Patterns は、human-guided AI 作業における反復する問題に対する再利用可能な解法を
収集します。Agentic coding はコンテキストファイル、ツール、テスト、リポジトリ、
レビューループ、永続的なアーティファクトが明確なため、最初の成熟したドメインです。
同じパターン言語は画像生成、文学・執筆、プロダクトデザイン、design-to-code、
video/media production、リサーチ、分析など、人間が AI を繰り返しワークフローで
導くあらゆるドメインにも適用されます。

これは software engineering の pattern-language の考え方を借用し、prompt、context
engineering、review loop、multi-agent orchestration、role contract、tool-use protocol に
適応させたものです。目標は、効果的な human-AI workflow をより簡単に命名、教育、批評、
自動化、改善できるようにすることです。

知的背景については [パターン言語のルーツ](foundations/pattern-language-roots.md) を参照してください。

## なぜ重要か

AI エージェントは膨大な数のパターンをマッチングできますが、依然として人間の目標、
判断、記憶、抽象化が必要です。人間はなぜあるワークフローが機能するかを形式的に
証明する前に、実用的なワークフローを発見します。そのワークフローにパターンとして
名前をつけると、伝達、検証、改善、再利用が容易になります。

このプロジェクトはパターンを運用知識の memetic unit（記憶・伝達可能な単位）として
扱います。すなわち、人間と AI システムが共通の目標に向けて協力するための、
コンパクトで伝達可能な解法です。

## プロジェクトスコープ

対象範囲:

- プロンプトと指示のパターン。
- コンテキストパケットとメモリパターン。
- 計画、実行、レビュー、検証のハーネス。
- マルチエージェントオーケストレーションパターン。
- ツール利用パターン。
- パターンスキーマとエージェント可読インデックス。
- Agentic coding、クリエイティブ生成、デザイン、執筆、リサーチ、その他反復する human-AI ワークフローのフィールドノートと証拠。

初期段階の対象外:

- 完全なホスト型パターンレジストリ。
- すべてのパターンの普遍的なランキングを主張するベンチマーク。
- ベンダー固有のマーケティングコンテンツ。
- 未整理の一般的 AI ニュースやプロンプト集。

## 初期読者

- Agentic coding を実践する開発者。
- ジェネラティブまたは agentic AI ワークフローを使用するデザイナー、ライター、リサーチャー、アーティスト、プロダクトビルダー、メディアクリエイター。
- Claude Code、Codex、Gemini などのエージェント指示文を書く人。
- ローカルエージェントワークフローの実践を文書化するメンテナー。
- Human-AI ソフトウェア作業を研究する実務家と研究者。

## シードステージの成功基準

- リポジトリが公開 GitHub リリースに十分なほど明確にプロジェクトを説明している。
- 少なくとも 3 つのパターンが望ましい品質基準を示している。
- 新しいコントリビューターが隠れたコンテキストなしでパターンを追加できる。
- エージェントが `AGENTS.md` を読んでカタログを一貫して維持できる。

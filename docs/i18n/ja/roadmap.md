# ロードマップ

このロードマップはシードステージのプロジェクトを集中させるための文書です。
コントリビューターがより良い証拠とより良いパターン名を持ち込むにつれて変わるべきものです。

## Phase 0: 言語の種を作る

- プロジェクトのタイトル、ミッション、スコープを定義する。
- パターンテンプレートと品質基準を確立する。
- 複数のカテゴリにまたがる小さなスターターカタログを追加する。
- エントリがすぐにドリフトしないよう軽量なバリデーションを追加する。

## Phase 1: Field Notes And Evidence

- 観察された agentic coding インシデントのための `field-notes/` 領域を拡張する。
- パターンを具体的なフィールドノートに紐付ける。
- 観察されたパターンと推測のパターンを区別する。
- `draft` から `candidate` への移行のための成熟度ルーブリックを追加する。

## Phase 2: Agent-Readable Documentation

- エージェント向けのコンパクトな公開パターンインデックスを生成する。
- エージェントが公開パターンドキュメントを読んで引用する例を追加する。
- 公開パッケージングはドキュメント、ナビゲーション、sanitized な例に集中する。

## Phase 3: Evaluation

- パターンの有用性を評価する実用的なメトリクスを定義する。
- 反復するタスククラスでパターンを比較する。
- 反例と deprecated なパターンを記録する。
- パターンエントリの承認または改訂のためのレビューワークフローを構築する。

## Phase 4: Public Community

- ライセンスを確定する。
- コントリビューションガバナンスを追加する。
- 最初の公開リリースを公開する。
- パターン提案と sanitized なフィールドノートを招待する。

## 近期バックログ

- 最終ライセンスを選択してルートの `LICENSE` ファイルまたは通知を追加する。
- パターン合成の例を追加する。
- エージェントが計画とレビュー中にパターンドキュメントを使用する公開例をさらに追加する。

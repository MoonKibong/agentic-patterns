# ハーネスパターン

Harness pattern は計画、実行、検証、批評、レビューの信頼性を高める段階的な
ワークフローを定義します。

単一の中断されない生成パスではタスクのリスクをカバーするには不十分な場合に使用します。

## 現在のパターン

- [生成-批評-改善 (Generate-Critique-Refine)](../../../../patterns/harness/generate-critique-refine.md): 候補を生成し、ルーブリックで批評した後、改善、選択、またはエスカレートします。

## パターンアイデア

- Red-Team Review.
- Reproduction First.
- Verification Ladder.
- Bounded Revision Loop.

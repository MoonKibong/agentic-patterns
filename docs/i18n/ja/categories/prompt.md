# プロンプトパターン

Prompt pattern はロール、タスク、制約、期待される出力のための再利用可能な指示構造を定義します。

主な問題がコードやツールの不足ではなく、エージェントの振る舞いの曖昧さにある場合に
使用します。つまり、何を最適化すべきか、何を決定できるか、何を返さなければならないかを
明確にします。

## 現在のパターン

- [ロール契約 (Role Contract)](../../../../patterns/prompt/role-contract.md): エージェントのロール、権限、責任、境界、出力、エスカレーションルールを定義します。

## パターンアイデア

- Reviewer Gate.
- Output Contract.
- Assumption Ledger.
- Decision Escalation.

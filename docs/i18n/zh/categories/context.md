# 上下文模式

Context pattern 定义如何为 agentic work 打包、精简、保留和移交信息。

当代理需要足够的信息才能正确行动，但直接倾倒文件、历史记录、日志或聊天内容又会掩盖重要信号时，请使用上下文模式。

## 当前模式

- [上下文包 (Context Packet)](../../../../patterns/context/context-packet.md)：在 planning、execution、review 或 handoff 前提供有界的任务上下文。
- [创意简报包 (Creative Brief Packet)](../../../../patterns/context/creative-brief-packet.md)：将 creative intent、constraint、reference、deliverable 和 evaluation criteria 打包用于生成性工作。

## 模式构想

- Constraint Header。
- Evidence Bundle。
- Resume Capsule。

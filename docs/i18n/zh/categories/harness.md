# 工作流模式

Harness pattern 定义分阶段的工作流，提高 planning、execution、verification、critique 和 review 的可靠性。

当任务风险足够高，单次不间断的生成过于薄弱时，请使用工作流模式。

## 当前模式

- [生成-批评-优化 (Generate-Critique-Refine)](../../../../patterns/harness/generate-critique-refine.md)：生成候选，用评分标准进行 critique，再 refine、select 或 escalate。

## 模式构想

- Red-Team Review。
- Reproduction First。
- Verification Ladder。
- Bounded Revision Loop。

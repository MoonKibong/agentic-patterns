# 路线图

本路线图旨在保持 seed-stage 项目的专注方向。随着贡献者带来更好的 evidence 和模式名称，路线图会随之更新。

## Phase 0：播下语言的种子

- 定义项目标题、使命和范围。
- 确立 pattern template 和质量标准。
- 在多个分类中添加初始 starter catalog。
- 添加轻量级验证，防止条目立即漂移。

## Phase 1：Field Notes And Evidence

- 扩展 `field-notes/` 区域，记录观察到的 agentic workflow incident。
- 将 pattern 与 field note 关联。
- 区分 observed pattern 与 speculative pattern。
- 添加从 `draft` 晋升到 `candidate` 的成熟度标准。

## Phase 2：Agent-Readable Documentation

- 生成供代理使用的 public pattern index。
- 添加代理读取并引用公开 pattern document 的示例。
- 公开发布内容专注于 documentation、navigation 和 sanitized example。

## Phase 3：Evaluation

- 定义衡量模式实用性的 practical metric。
- 在重复任务类别中比较模式。
- 记录反例和已废弃的模式。
- 构建接受或修订模式条目的审查工作流。

## Phase 4：Public Community

- 准备公开发布。
- 添加贡献治理机制。
- 发布首个 public release。
- 邀请模式提案和 sanitized field note。

## 近期待办

- 添加模式组合示例。
- 添加代理在 planning 和 review 过程中使用 pattern document 的公开示例。

# Hoja de Ruta

Esta hoja de ruta es un documento para enfocar el proyecto en etapa semilla. Puede cambiar
a medida que llegue mejor evidencia y mejores nombres de patrones.

## Fase 0: Crear la Semilla del Lenguaje

- Definir el título, la misión y el alcance del proyecto.
- Establecer la plantilla de patrón y el nivel de calidad.
- Añadir un catálogo inicial con patrones en varias categorías.
- Añadir validación ligera para prevenir la deriva de entradas.

## Fase 1: Notas de Campo y Evidencia

- Expandir el área `field-notes/` para incidentes de flujos de trabajo agénticos observados.
- Conectar patrones con notas de campo.
- Distinguir patrones observados de patrones especulativos.
- Añadir una rúbrica de madurez para pasar de `draft` a `candidate`.

## Fase 2: Documentación Legible por Agentes

- Crear un índice de patrones públicos para agentes.
- Añadir ejemplos de agentes que lean y citen documentos de patrones públicos.
- El empaquetado público se centra en documentación, navegación y ejemplos sanitizados.

## Fase 3: Evaluación

- Definir métricas prácticas para juzgar la utilidad de los patrones.
- Comparar patrones en clases de tareas repetidas.
- Registrar contraejemplos y patrones deprecados.
- Construir un flujo de trabajo de revisión para aceptar o revisar entradas de patrones.

## Fase 4: Comunidad Pública

- Preparar el lanzamiento público.
- Añadir gobernanza de contribuciones.
- Publicar el primer lanzamiento público.
- Invitar propuestas de patrones y notas de campo sanitizadas.

## Backlog a Corto Plazo

- Añadir ejemplos de composición de patrones.
- Añadir ejemplos públicos de agentes que usen documentos de patrones durante la planificación y revisión.

# Patterns de contexte

Les patterns de contexte définissent comment empaqueter, réduire, préserver et transmettre les
informations pour le travail agentique.

Utilisez les patterns de contexte lorsqu'un agent a besoin de suffisamment d'informations pour agir
correctement, mais qu'un déversement brut de fichiers, d'historique, de logs ou de chat cacherait
le signal important.

## Patterns actuels

- [Paquet de contexte (Context Packet)](../../../../patterns/context/context-packet.md) : fournir un
  contexte borné et orienté tâche avant la planification, l'exécution, la révision ou le transfert.
- [Paquet de brief créatif (Creative Brief Packet)](../../../../patterns/context/creative-brief-packet.md) :
  empaqueter l'intention créative, les contraintes, les références, les livrables et les critères
  d'évaluation pour le travail génératif.

## Idées de patterns

- Handoff Summary.
- Constraint Header.
- Evidence Bundle.
- Resume Capsule.

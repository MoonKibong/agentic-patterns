# Patterns de prompts

Les patterns de prompts définissent des structures d'instructions réutilisables pour les rôles, les
tâches, les contraintes et les sorties attendues.

Utilisez les patterns de prompts lorsque le problème principal n'est pas un code manquant ou des
outils manquants, mais un comportement peu clair : ce que l'agent doit optimiser, ce qu'il peut
décider et ce qu'il doit retourner.

## Patterns actuels

- [Contrat de rôle (Role Contract)](../../../../patterns/prompt/role-contract.md) : définir un rôle
  d'agent à travers l'autorité, les responsabilités, les limites, les sorties et les règles
  d'escalade.

## Idées de patterns

- Reviewer Gate.
- Output Contract.
- Assumption Ledger.
- Decision Escalation.

# 프로젝트 개요

## 작업 제목

**Agentic Patterns: A Pattern Language for Human-Guided AI Work**

현재 공개 제목으로 가장 적합한 후보입니다. `Agentic Patterns`라는 짧고 기억하기 쉬운
이름을 유지하면서 더 큰 목표를 설명합니다.

## 대체 제목

- **Agentic Patterns: Reusable Context for Human-Guided AI**
- **Agentic Patterns: A Catalog of Prompts, Context, and Harnesses**
- **The Agentic Pattern Language**
- **Agentic Patterns: Reusable Knowledge for AI-Assisted Work**

가장 강한 방향은 첫 제목입니다. 이 프로젝트는 단순한 prompt 모음이 아니라, 사람이
AI 에이전트를 지휘하기 위해 사용하는 언어, 컨텍스트, 도구, 프로토콜에 관한 것입니다.

## 한 문장 설명

Agentic Patterns는 반복되는 창작, 기술, 지식 작업에서 사람이 AI 에이전트를 더 잘
지휘할 수 있도록 재사용 가능한 prompt, context structure, harness, role contract, protocol을
정리하는 공개 카탈로그이자 패턴 언어입니다.

## 짧은 설명

Agentic Patterns는 human-guided AI 작업에서 반복적으로 나타나는 문제에 대한 재사용
가능한 해법을 수집합니다. Agentic coding은 컨텍스트 파일, 도구, 테스트, 저장소,
리뷰 루프, 지속 가능한 아티팩트가 명확하기 때문에 첫 번째 성숙한 도메인입니다.
동일한 패턴 언어는 이미지 생성, 글쓰기, 제품 디자인, design-to-code, video/media
production, research, analysis에도 적용됩니다.

이 프로젝트는 software engineering의 pattern-language 아이디어를 빌려와 prompt, context
engineering, review loop, multi-agent orchestration, role contract, tool-use protocol에
적용합니다. 목표는 효과적인 human-AI workflow를 더 쉽게 명명, 교육, 비평, 자동화, 개선하는
것입니다.

지적 배경은 [패턴 언어의 뿌리](foundations/pattern-language-roots.md)를 참고하세요.

## 왜 중요한가

AI 에이전트는 엄청난 수의 패턴을 매칭할 수 있지만, 여전히 인간의 목표, 판단, 기억,
추상화가 필요합니다. 인간은 왜 어떤 흐름이 작동하는지 완전히 증명하기 전에 실용적인
워크플로를 발견합니다. 그 워크플로에 이름을 붙이면 전달, 검증, 개선, 재사용이 쉬워집니다.

이 프로젝트는 패턴을 운영 지식의 memetic unit으로 다룹니다. 즉, 사람이 AI와 공통
목표를 향해 협력하기 위한 작고 전달 가능한 해법입니다.

## 프로젝트 범위

포함:

- Prompt와 instruction pattern.
- Context packet과 memory pattern.
- Planning, execution, review, verification harness.
- Multi-agent orchestration pattern.
- Tool-use pattern.
- Pattern schema와 agent-readable index.
- Coding, creative generation, design, writing, research 등 반복되는 human-AI workflow의 field note와 evidence.

초기 단계에서 제외:

- 전체 hosted pattern registry.
- 모든 패턴의 보편적 순위를 주장하는 benchmark.
- Vendor-specific marketing content.
- 정리되지 않은 일반 AI 뉴스나 prompt 모음.

## 초기 독자

- Agentic coding을 실천하는 개발자.
- Generative 또는 agentic AI workflow를 사용하는 designer, writer, researcher, artist, product builder, media creator.
- Claude Code, Codex, Gemini 등 에이전트 지시문을 작성하는 사람.
- 로컬 agent workflow practice를 문서화하는 maintainer.
- Human-AI software work를 연구하는 실무자와 연구자.

## Seed-stage 성공 기준

- 공개 GitHub release가 가능할 정도로 프로젝트 설명이 명확하다.
- 최소 세 개 이상의 패턴이 원하는 품질 기준을 보여준다.
- 새 기여자가 숨은 맥락 없이 패턴을 추가할 수 있다.
- 에이전트가 `AGENTS.md`를 읽고 일관되게 카탈로그를 유지할 수 있다.

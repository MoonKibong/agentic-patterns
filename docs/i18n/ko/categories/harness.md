# 하네스 패턴

Harness pattern은 planning, execution, verification, critique, review의 신뢰도를 높이는 staged
workflow를 정의합니다.

Single uninterrupted generation pass가 위험할 정도로 task risk가 있을 때 사용합니다.

## 현재 패턴

- [생성-비평-개선 (Generate-Critique-Refine)](../../../../patterns/harness/generate-critique-refine.md): candidate를 생성하고 rubric으로 critique한 뒤 refine, select, escalate합니다.

## 패턴 아이디어

- Red-Team Review.
- Reproduction First.
- Verification Ladder.
- Bounded Revision Loop.

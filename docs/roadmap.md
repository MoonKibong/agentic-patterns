# Roadmap

This roadmap keeps the seed-stage project focused. It should change as contributors bring better
evidence and better pattern names.

## Phase 0: Seed The Language

- Define the project title, mission, and scope.
- Establish the pattern template and quality bar.
- Add a small starter catalog across several categories.
- Add lightweight validation so entries do not drift immediately.

## Phase 1: Field Notes And Evidence

- Add a `field-notes/` area for observed agentic coding incidents.
- Link patterns to concrete field notes.
- Distinguish observed patterns from speculative patterns.
- Add a pattern maturity rubric for moving from `draft` to `candidate`.

## Phase 2: Agent-Readable Documentation

- Generate compact public pattern indexes for agents.
- Add examples showing how agents can read and cite public pattern documents.
- Keep public packaging focused on documentation, navigation, and sanitized examples.

## Phase 3: Evaluation

- Define practical metrics for pattern usefulness.
- Compare patterns on repeated task classes.
- Record counterexamples and deprecated patterns.
- Build review workflows for accepting or revising pattern entries.

## Phase 4: Public Community

- Finalize license.
- Add contribution governance.
- Publish the first public release.
- Invite pattern proposals and sanitized field notes.

## Near-Term Backlog

- Choose final license and add root `LICENSE` files or notices.
- Add examples of pattern composition.
- Add more public examples of agents using pattern documents during planning and review.

# Evaluation Patterns

Evaluation patterns define how humans and agents judge whether a pattern, plan, workflow, or output
is appropriate and effective.

Use evaluation patterns when the central question is not "what should the agent do?" but "how do we
know this approach fits or worked?"

## Current Patterns

No public evaluation pattern is currently listed. Private evaluation patterns are maintained outside
the public wiki.

## Pattern Ideas

- Evidence Strength.
- Pattern Maturity.
- Counterexample Review.
- Outcome Trace.

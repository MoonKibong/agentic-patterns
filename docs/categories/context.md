# Context Patterns

Context patterns define how to package, reduce, preserve, and hand off information for agentic work.

Use context patterns when an agent needs enough information to act correctly, but a raw dump of files,
history, logs, or chat would hide the important signal.

## Current Patterns

- [Context Packet](../../patterns/context/context-packet.md): provide bounded, task-shaped context
  before planning, execution, review, or handoff.
- [Creative Brief Packet](../../patterns/context/creative-brief-packet.md): package creative intent,
  constraints, references, deliverables, and evaluation criteria for generative work.

## Pattern Ideas

- Handoff Summary.
- Constraint Header.
- Evidence Bundle.
- Resume Capsule.

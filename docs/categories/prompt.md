# Prompt Patterns

Prompt patterns define reusable instruction structures for roles, tasks, constraints, and expected
outputs.

Use prompt patterns when the main problem is not missing code or missing tools, but unclear behavior:
what the agent should optimize for, what it may decide, and what it must return.

## Current Patterns

- [Role Contract](../../patterns/prompt/role-contract.md): define an agent role through authority,
  responsibilities, boundaries, outputs, and escalation rules.

## Pattern Ideas

- Reviewer Gate.
- Output Contract.
- Assumption Ledger.
- Decision Escalation.

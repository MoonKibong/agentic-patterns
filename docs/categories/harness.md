# Harness Patterns

Harness patterns define staged workflows that improve reliability around planning, execution,
verification, critique, and review.

Use harness patterns when a task has enough risk that a single uninterrupted generation pass is too
weak.

## Current Patterns

- [Generate-Critique-Refine](../../patterns/harness/generate-critique-refine.md): generate
  candidates, critique with a rubric, refine, and select or escalate.

## Pattern Ideas

- Red-Team Review.
- Reproduction First.
- Verification Ladder.
- Bounded Revision Loop.
